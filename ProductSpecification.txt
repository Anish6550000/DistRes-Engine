1. Overview
-----------
The Distributed Resource Access and Synchronisation Engine (DistRes) extends the
ConRes (v1) concurrent engine into a distributed client-server system. Multiple
client nodes connect to a single server node over TCP and share safe, coordinated
read/write access to this specification file and the user-credential database.

2. Requirements
---------------
- Distributed client-server communication over TCP sockets
- Concurrent reads, single exclusive writer (read-write coordination)
- Publish-subscribe notification of every committed write to all active clients
- Layered architecture separating application/logic and data layers
- Distributed fault tolerance via automatic reconnection and retries
